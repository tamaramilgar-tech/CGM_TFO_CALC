Cómo está pensada

Estructura didáctica de cada fase

1. Situación real

El alumnado parte de un encargo cercano al entorno administrativo: ventas, notas, informes, clientes, inventario o intercambio de archivos.

2. Desarrollo guiado

Se indica qué introducir en cada columna, qué fórmulas escribir y qué resultado debería aparecer para poder comprobar si el trabajo está bien hecho.

3. Comprobación final

Cada fase termina con 10 preguntas tipo test de 4 opciones, útiles para repasar conceptos o hacer evaluación rápida.

Itinerario didáctico

Las 6 fases del módulo

Una práctica completa por cada bloque de contenido solicitado.

Fase A

Tipos de datos y referencias

Datos, celdas, rangos, hojas, referencias relativas, absolutas y mixtas en una hoja de ventas.

 Fase B

Fórmulas y funciones

Operaciones y funciones en el cálculo de calificaciones y estadísticas de grupo.

 Fase C

Gráficos

Creación de gráficos de columnas, líneas y sectores para un informe de ventas trimestrales.

 Fase D

Macros y automatización

Grabación y uso de macros para dar formato uniforme y preparar plantillas repetitivas.

 Fase E

Importación y exportación

Trabajo con CSV, XLSX y ODS para abrir, adaptar y compartir hojas creadas con otras aplicaciones.

 Fase F

Calc como base de datos

Listas, formularios, filtrado, ordenación y protección sobre una base de datos de clientes.

Uso recomendado en clase

Propuesta de secuencia

  1. Presentar el caso real y los conceptos previos.
  2. Resolver en gran grupo los dos o tres primeros pasos.
  3. Dejar que el alumnado continúe siguiendo la guía.
  4. Comparar el resultado obtenido con la solución esperada.
  5. Finalizar con el test para consolidar el aprendizaje.

Qué encontrará el alumnado

Datos cerrados: no necesitan inventar la tabla inicial.
Pasos claros: qué escribir, dónde y para qué.
Autocontrol: pueden revisar si el resultado coincide.
Navegación sencilla: cada fase tiene acceso a la siguiente.
